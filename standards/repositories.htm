iterami/Docs.htm/Standards/Repositories
  • Settings:
    • Actions are disabled.
    • GitHub Pages is used whenever possible.
      • HTTPS is enforced.
    • Issues are enabled.
    • Labels that aren't "Bug" or "Enhancement" are removed. Label descriptions are removed.
    • Projects are disabled.
    • Pull Requests are enabled.
    • Repository names should include additional information to indicate their project types:
      Type Prefix Suffix Topics
      Atom atom, atom-package
      Assembly .asm
      C++ .cpp
      Chromium Chromium- chromium-browser
      Diablo D2- or D2LoD-
      Firefox Firefox- firefox
      GTK .gtk gtk3
      HTM and GitHub Pages .htm
      JavaScript .js
      Minecraft MC- minecraft
      PHP .php
      Python .py
      Shell .sh
      Starcraft SC- starcraft
      Warzone 2100 WZ2100- warzone2100
      World of Warcraft WoW world-of-warcraft
    • Sponsorships are enabled.
    • Template repository setting should be enabled for every repository.
    • Topics should be used but limited. Must be in alphabetical order.
    • Wikis are disabled and restricted to users with push access.
  • Creating a new GitHub Pages repository:
    1. Locally prepare the first repository commit.
    2. Create the repository on GitHub and make sure its settings match the list in the settings section.
    3. Clone the repository locally, add the locally prepared project files, and push them to the "master" branch.
    4. Run gh-pages-from-master.sh on the cloned repository. Make "gh-pages" the default branch on GitHub and delete the "master" branch.
    5. Add the repository link to various iterami repositories:
      • Repositories table so that docs, common file usage, and tests can be managed.
      • iterami.github.io so that it may have a link on the index page.
      • RandomRepository.htm so that it may be randomly selected.
      • Scripts iterami-repositories-list.sh so that it may be managed in bulk.
    6. Update the repository "Description" with an emoji, mention of CC0 license, description of project and type, and a link to it hosted on https://iterami.github.io.
    7. Update the repository "Website" with a link to it hosted on https://iterami.github.io.
    8. Update the repository "Topics" with at least one topic.
  • Creating a new repository not hosted on GitHub Pages:
    1. Locally prepare the first repository commit.
    2. Create the repository on GitHub and make sure its settings match the list in the settings section.
    3. Clone the repository locally, add the locally prepared project files, and push them to the "master" branch.
    4. Add the repository link to various iterami repositories:
      • Repositories table so that docs, common file usage, and tests can be managed.
      • iterami.github.io so that it may have a link on the index page.
      • RandomRepository.htm so that it may be randomly selected.
      • Scripts iterami-repositories-list.sh so that it may be managed in bulk.
    5. Update the repository "Description" with an emoji, mention of CC0 license, description of project and type, and a link to download the repository as a ".tar.gz" file.
    6. Update the repository "Website" with a link to download the repository as a ".tar.gz" file.
    7. Update the repository "Topics" with at least one topic.
