Priorities:
  1. Discussions and pull requests to solve open issues.
  2. Code additions and improvements to core iterami repos:
    1. iterami multiverse repos need additions and improvements:
      1. iterami/Multiverse.htm
      2. iterami/MultiverseEditor.htm
      3. iterami/MultiverseViewer.htm
    2. iterami/common needs code additions/improvements.
    3. iterami/Docs.htm needs additional and improved docs for all repos.
  3. Code additions and improvements to all other iterami repos, all with equal priority:
    • iterami/Guides.htm needs more information about existing topics, as well as new topics.
    • iterami/UnitConverter.htm needs more units and current conversions to be verified.
  4. Check for changes that may require repo updates:
    EVE-Overviews.yaml https://eveonline.com/news/t/patch-notes
    GMod-Addons.lua https://store.steampowered.com/news/app/4000
    MC-DataPacks https://minecraft.fandom.com/wiki/Java_Edition_version_history
    WoW-Addons.lua https://worldofwarcraft.com/en-us/content-update-notes
    WZ2100-AI.js https://github.com/Warzone2100/warzone2100/blob/master/ChangeLog
Issues:
  • Issues can be made about anything.
  • Issues will not be assigned to anybody.
  • Labels are limited to "Bug" and "Enhancement". All other labels are removed.
  • If your issue is about something handled by iterami/common, it should be reported in that repo instead of in a repo that only utilizes the common code.
  • Comments can be added to closed issues.
Pull Requests:
  • Pull requests can be made about anything.
  • Pull requests will not be assigned to anybody.
  • Pull requests will not be labled.
  • Make sure your contribution complies with the license choice and code standards of the repo you wish you contribute to.
