iterami/Docs.htm/Standards/Contributing
  • Priorities:
    1. Discussions and pull requests to solve open issues.
    2. Code additions and improvements to core iterami repositories:
      1. iterami multiverse repositories need code additions/improvements and additional levels:
        1. iterami/Multiverse.htm
        2. iterami/MultiverseEditor.htm
        3. iterami/MultiverseLevels.htm
      2. iterami/common needs code additions/improvements.
      3. iterami/Docs.htm needs additional and improved docs for all repositories.
    3. Code additions and improvements to all other iterami repositories, all with equal priority:
      • iterami/Guides.htm needs more information about existing topics, as well as new topics.
      • iterami/UnitConverter.htm needs more units and current conversions to be verified.
  • Issues:
    • Issues can be made about anything.
    • Issues will not be assigned to anybody.
    • Labels are limited to "Bug" and "Enhancement". All other labels are removed.
    • If your issue is about something handled by iterami/common, it should be reported in that repository instead of in a repository that only utilizes the common code.
  • Pull Requests:
    • Pull requests can be made about anything.
    • Pull requests will not be assigned to anybody.
    • Pull requests will not be labled.
    • Make sure your contribution complies with the license choice and code standards of the repository you wish you contribute to.
