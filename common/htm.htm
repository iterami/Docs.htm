Recommended Project Structure:
  • /js/:
    • data.js: contains repo-specific code, such as levels or custom functions.
    • main.js: main project JavaScript file, which only contains functions defined by iterami/common.
  • 404.htm: handles 404 errors for GitHub Pages repos.
  • index.htm: the page on which your project resides.
  • LICENSE.md: contains license information about your project.
  • README.md: contains information about your project.
Using and Updating:
  • If your project is being hosted on GitHub Pages, you should use a fork of iterami/common:
    • Update your fork by using the iterami/Scripts/iterami/common-fork-rebase.sh script.
  • If your project is being hosted on a different server, you should upload the iterami/common files:
    • You do not need to upload the .git folder.
    • Clone iterami/common via git. Pull and upload any changed files to your server.
    • Your iterami/common folder does not need to be in a particular location, as long as the paths in your <script> tags are correct.
