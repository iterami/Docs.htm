iterami/Docs.htm/common HTM Projects
  • CSS Files:
    • canvas.css: used by 2D canvas and 3D WebGL HTM projects.
    • colors.css: used by HTM projects that require specific colors with special meanings.
    • core.css: used by every HTM project that isn't only text.
    • dom.css: used by HTM projects that use buttons to simulate a canvas.
    • writings.css: used by HTM projects that are only text.
  • JavaScript Files:
    • audio.js: used by HTM projects that require audio.
    • canvas.js: used by 2D canvas HTM projects.
    • core.js: used by every HTM project.
    • entity.js: used by HTM projects that require entities and groups.
    • math.js: used by HTM projects that require advanced math.
    • tables.js: stand-alone file that adds table column sorting.
    • test.js: used by HTM projects that want to test functions and performance.
    • time.js: used by HTM projects that utilize dates, timestamps, and time formatting.
    • webgl.js: used by 3D WebGL HTM projects.
    • Prefabs:
      • canvas.js: prefabricated 2D canvas entities.
      • uri.js: prefabricated data URIs.
      • webgl.js: prefabricated 3D WebGL entities.
  • Recommended Project Structure:
    • /js/:
      • data.js: contains repository-specific code, such as levels or custom functions.
      • main.js: main project JavaScript file, which only contains functions defined by iterami/common.
    • 404.htm: handles 404 errors for GitHub Pages repositories.
    • index.htm: the page on which your project resides.
    • LICENSE.md: contains license information about your project.
    • README.md: contains information about your project.
  • Using and Updating:
    • If your project is being hosted on GitHub Pages, you should use a fork of iterami/common:
      • Update your fork by using the iterami/Scripts/iterami/common-fork-rebase.sh script.
    • If your project is being hosted on a different server, you should upload the iterami/common files:
      • You do not need to upload the .git directory.
      • Clone iterami/common via git. Pull and upload any changed files to your server.
      • Your iterami/common directory does not need to be in a particular location, as long as the paths in your <script> tags are correct.
